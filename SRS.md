<h2 id="introducere">1. Introducere</h2>
<h3 id="scopul-documentului">1.1 Scopul documentului</h3>
<p>Acest document are ca scop descrierea specificațiilor cerințelor software pentru platforma KnowledgeKey. Acest document face trecerea de la ce trebuie realizat la cum poate fi modelat sistemul în vederea proiectării. Documentul pune la dispoziție trei perspective legate de informație, funcționalități și comportamente și este baza pentru proiectarea lucrării de licență.</p>
<h3 id="domeniul-de-aplicație">1.2 Domeniul de aplicație</h3>
<p>Această platformă de e-learning este destinată instituțiilor, licee și universități. Aceasta oferă management pentru cursuri și utilizatori.</p>
<p>Studenții pot accesa materiale, își pot seta singuri ritmul de învățare, pot folosi zona de antrenament pentru pregătirea în vederea examenelor. Se pot organiza singuri și își pot manegeria timpul cu ajutorul unui timer de tipul Pomodoro. Pentru o organizare optimă, platforma are și un calendar unde studenții pot vedea temele cu termenele limită aferente și își pot organiza materia într-un notebook personal.</p>
<p>Profesorii pot crea teste personalizate, pot adăuga teme și materiale. La adăugarea fiecărei teme sau examen, acestea vor fi adăugate automat în calendar. Sistemul permite crearea de examene unice pentru fiecare elev. Profesorii pot adăuga materiale ajutătoare, cum ar fi hinturile.</p>
<p>Pentru motivarea studenților, există și un modul de gamificare, unde pot câștiga puncte pentru utilizarea aplicației zilnic sau la o perioadă predeterminată de profesor.</p>
<p>În timpul examenelor și al testelor de antrenament, sistemul urmărește timpul petrecut pentru fiecare întrebare, de câte ori un student a cerut ajutor prin hinturi și de câte ori a sărit peste anumite întrebări. Aceste metrice ajută la determinarea dificultății întrebărilor și adaptarea examenelor viitoare.</p>
<p>Profesorii pot vedea, într-un panou dedicat aceste date, alături de rapoarte cuprinzătoare pentru rezultatele fiecărui student în parte, dar și pentru clasele de studenți.</p>

<h3 id="perspectiva-produsului">2.1 Perspectiva produsului</h3>
<p>Knowledge Key este un sistem software care funcționează ca platformă web accesibilă prin browser.</p>
<ul>
<li><strong>Backend:</strong>  Django REST Framework (Python) - gestionează logica, autentificarea și comunicarea cu baza de date</li>
<li><strong>Frontend:</strong>  template-uri Django, CSS  și JavaScript pentru funcționalități precum FullCalendar, SortableJS, timer Pomodoro.</li>
<li><strong>Baza de date:</strong>  SQLite pentru dezvoltare, MongoDB pentru producție</li>
<li><strong>Storage:</strong>  Google Cloud Storage pentru fișiere media</li>
</ul>
<h3 id="funcții-de-ansamblu">2.2 Funcții de ansamblu</h3>
<ul>
<li>Autentificare și roluri (Student, Teacher, Admin).</li>
<li>Management utilizatori și instituții (CRUD, import CSV).</li>
<li>Cursuri, materiale (PDF, video, link), anunțuri, module microlearning.</li>
<li>Zona de învățare: Pomodoro/study timer, notebook personal, progres pe module.</li>
<li>Teste și practice: întrebări avansate, hint-uri pe niveluri, EmotionEngine (practice adaptiv), exam builder (drag &amp; drop), generare teste unice, reguli flexibile de notare.</li>
<li>Teme (assignments), milestones, rubrici de evaluare.</li>
<li>Gamificare: puncte, insigne, streak-uri</li>
<li>Calendar și remindere; generare automată evenimente pentru teme și teste.</li>
<li>SRL (obiective, jurnal, reflecții), proctoring soft, rapoarte avansate, API REST.</li>
</ul>
<h3 id="clase-de-utilizatori-și-caracteristici">2.3 Clase de utilizatori și caracteristici</h3>
<p><strong>Admin</strong>: Administrator platformă → Gestionare instituții, profesori, studenți; import CSV; asignare cursuri; acces admin Django.<br>
<strong>Profesor</strong>: Cadru didactic → Creează/editează cursuri, materiale, anunțuri; module microlearning; întrebări și teste; exam builder; teme și rubrici; vede dashboard cu statistici; poate vedea rapoarte.<br>
<strong>Student</strong>:Elev / student → Vizualizează cursuri și materiale; parcurge module; folosește Pomodoro, notebook, calendar; rezolvă practice și teste; trimite teme; vede puncte și insigne; (viitor) SRL.</p>
<h2 id="cazuri-de-utilizare-use-cases">3. Cazuri de utilizare (use cases)</h2>
<p>Setul de <strong>cazuri de utilizare de bază</strong> descrie interacțiunile principale ale actorilor cu sistemul. Fiecare caz include: actor, scop, precondiții, flux principal, fluxuri alternative și postcondiții.</p>
<h3 id="uc-01-autentificare-login">3.1 UC-01: Autentificare (Login)</h3>
<p><strong>Actor principal</strong>: Utilizator (Student, Profesor sau Admin)</p>
<p><strong>Scop</strong>: Accesarea contului pentru a folosi funcționalitățile rolului respectiv.</p>
<p><strong>Precondiții</strong>: Utilizatorul are cont creat în sistem; nu este deja autentificat.</p>
<p><strong>Flux principal</strong>:</p>
<ol>
<li>Utilizatorul deschide pagina de login.</li>
<li>Introdu username și parola.</li>
<li>Apasă „Login”.</li>
<li>Sistemul validează credentialele.</li>
<li>Sistemul creează sesiune și redirecționează la dashboard-ul corespunzător rolului (admin / profesor / student).</li>
</ol>
<p><strong>Flux alternativ</strong>:<br>
4a. Credentiale incorecte: sistemul afișează mesaj de eroare și rămâne pe pagina de login.</p>
<p><strong>Postcondiții</strong>: Utilizatorul este autentificat; sesiunea este activă.</p>
<h3 id="uc-02-vizualizare-curs-și-materiale-student">3.2 UC-02: Vizualizare curs și materiale (Student)</h3>
<p><strong>Actor principal</strong>: Student</p>
<p><strong>Scop</strong>:  Consultarea cursurilor la care este înscris și a materialelor (PDF, video, link) puse la dispoziție.</p>
<p><strong>Precondiții</strong>: Studentul este autentificat; este înscris la cel puțin un curs.</p>
<p><strong>Flux principal</strong>:</p>
<ol>
<li>Studentul accesează dashboard-ul și selectează un curs.</li>
<li>Sistemul afișează detaliile cursului și lista de materiale.</li>
<li>Studentul deschide un material (descărcare PDF, vizionare video sau link).</li>
<li>Studentul poate consulta și anunțurile afișate pentru curs.</li>
</ol>
<p><strong>Flux alternativ</strong>:<br>
2a. Cursul nu are materiale: se afișează mesaj corespunzător.</p>
<p><strong>Postcondiții</strong>:<br>
Sunt afișate materialele și anunțurile pentru curs.</p>
<h3 id="uc-03-parcurgere-modul-microlearning-student">3.3 UC-03: Parcurgere modul microlearning (Student)</h3>
<p><strong>Actor principal</strong>: Student</p>
<p><strong>Scop</strong>: Parcurgerea unui modul de microlearning și marcarea acestuia ca completat.</p>
<p><strong>Precondiții</strong>: Studentul este autentificat; este înscris la cursul care conține modulul; modulul este publicat.</p>
<p><strong>Flux principal</strong>:</p>
<ol>
<li>Studentul deschide lista de module a cursului.</li>
<li>Selectează un modul (nou sau „continuă unde ai rămas”).</li>
<li>Sistemul afișează conținutul modulului (video, text, quiz etc.) în ordine.</li>
<li>Studentul parcurge conținutul; sistemul poate actualiza progresul (ex.: procent) la cerere.</li>
<li>La final, studentul apasă „Marchează ca completat”.</li>
<li>Sistemul salvează ModuleProgress (completed_at, progress 100%) și acordă puncte de gamificare (dacă este cazul).</li>
</ol>
<p><strong>Flux alternativ</strong>:<br>
5a. Studentul părăsește fără a marca completat: progresul parțial rămâne salvat.</p>
<p><strong>Postcondiții</strong>:<br>
Progresul modulului este actualizat; la completare, studentul primește puncte.</p>
<h3 id="uc-04-rezolvare-test-student">3.4 UC-04: Rezolvare test (Student)</h3>
<p><strong>Actor principal</strong>: Student</p>
<p><strong>Scop</strong>: Rezolvarea unui test și obținerea unei note.</p>
<p><strong>Precondiții</strong>: Studentul este autentificat; testul este disponibil pentru cursul la care este înscris; nu a depășit numărul permis de încercări (dacă există limitare).</p>
<p><strong>Flux principal</strong>:</p>
<ol>
<li>Studentul deschide testul (sau testul generat unic pentru el).</li>
<li>Sistemul afișează întrebările una câte una.</li>
<li>Studentul răspunde la fiecare întrebare (poate solicita hint-uri dacă sunt permise).</li>
<li>Studentul trimite testul.</li>
<li>Sistemul notează răspunsurile (FlexibleScorer), salvează Attempt și UserAnswer.</li>
<li>Sistemul afișează rezultatul (scor, trecut/nepromovat).</li>
<li>La trecere/perfect, sistemul acordă puncte de gamificare.</li>
</ol>
<p><strong>Flux alternativ</strong>:<br>
3a. Studentul cere un hint: sistemul dezvăluie nivelul de hint solicitat și poate înregistra QuestionHintUsage.<br>
4a. Timp expirat (dacă există limită): sistemul închide automat și notează ce a fost răspuns.</p>
<p><strong>Postcondiții</strong>: Attempt și răspunsurile sunt salvate; studentul vede rezultatul; punctele sunt acordate conform regulilor.</p>
<h3 id="uc-05-trimitere-temă-student">3.5 UC-05: Trimitere temă (Student)</h3>
<p><strong>Actor principal</strong>: Student<br>
<strong>Scop</strong>: Încărcarea unei teme (fișier/link) înainte de deadline.</p>
<p><strong>Precondiții</strong>: Studentul este autentificat; tema există și este activă; deadline-ul nu a trecut.</p>
<p><strong>Flux principal</strong>:</p>
<ol>
<li>Studentul accesează lista de teme (din dashboard sau calendar).</li>
<li>Selectează tema și deschide detaliul.</li>
<li>Încarcă fișierul sau introduce link-ul conform cerințelor temei.</li>
<li>Apasă „Trimite”.</li>
<li>Sistemul salvează Submission și afișează confirmare.</li>
</ol>
<p><strong>Flux alternativ</strong>:<br>
4a. Format invalid sau lipsă fișier: sistemul afișează eroare și nu salvează.<br>
4b. Deadline depășit: sistemul refuză trimiterea și afișează mesaj.</p>
<p><strong>Postcondiții</strong>: Submission este creat; profesorul poate nota tema.</p>
